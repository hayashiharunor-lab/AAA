# HR-MANAGER

教育機関における生徒育成・教育管理を中心とした管理システム。

## 開発環境

- PHP
- MySQL
- HTML/CSS
- PDO接続

---

## 基本思想

> 必要な情報を、必要な人に、必要なタイミングで表示する。

大量のメニューを最初から表示しない。現場の先生が迷わず使えることを最優先にする。

### 家を建てるように開発する

- 基礎を作る
- 柱を作る
- 部屋を作る
- 不要な型枠(仮設コード・不要フォルダ)は撤去する

完成後に不要なものを残さない。

---

## 開発ルール

### 1. シンプル優先

裏側は格好良さより、「初心者でも理解できる」「未来の自分が読める」ことを優先する。

### 2. ボタンは固定で書かない

先生A・先生B…と直接書くのではなく、データベース登録 → 自動表示にする。
対象:学校 / クラス / 担当者 / その他マスター

### 3. マスターと業務を分離する

- マスター(設定情報):学校 / 担当者 / クラス
- 業務(日々入力するもの):日報 / 成績 / 出欠 / 授業記録

---

## トップメニュー設計

- 生徒管理
- 教育管理
- 外部連携管理
- マスター管理(管理者のみ)

## 生徒管理

在学中の生徒管理を担当。卒業後は卒業履歴として保存し、必要なら別の人材管理システムへ橋渡しする(HR-MANAGER内では卒業後の人材管理は行わない)。

## 教育管理

学校管理という独立メニューは廃止し、教育管理の中で扱う。

教育管理 → 学校選択 → クラス選択 → 担当者選択 → 教育業務

クラス中心設計:先生と生徒を直接紐付けしない(先生の異動・変更に対応するため)。

学校 → クラス → 担当者 → 生徒

## 担当者設計

「先生選択」ではなく「担当者選択」と呼ぶ。入力責任者(誰が入力したか・誰が確認責任を持つか)を記録するため。
教育分析では入力者ではなく「クラス・授業担当・教科」を基準にする。

## 外部連携管理

提携教育機関・受入企業を管理。将来的に企業担当者の名刺AI読み込み構想あり。
